# keyboard.tsx 菜单自动刷新问题修复说明

## 问题描述
用户在 Memo tab 长按 memo 弹出快捷菜单后，滑动查看菜单项时，约 1 秒后菜单会自动刷新回初始位置。

## 根本原因

### 问题1：setInterval 中的条件判断错误
在 `KeyboardView.tsx` 的 `setInterval` 定时器中，条件判断使用了 `scopeTabForKeyboardRefresh(activeKeyboardScope)` 而不是 `activeTab`，导致即使当前在 Memo tab 上，也会触发剪贴板数据刷新。

### 问题2：useEffect 依赖数组为空
`setInterval` 所在的 `useEffect` 有空的依赖数组 `[]`，所以只在组件挂载时运行一次。其中的 `activeTab` 值是初始值，不会随标签切换而更新。

### 问题3：refresh 函数强制刷新
`setInterval` 中调用 `refresh(true, ...)` 时，`force = true` 会跳过内容比较，每次都调用 `setItems`，即使内容没有变化也会触发重新渲染。

### 问题4：非剪贴板tab上的不必要状态更新
`setInterval` 每 900ms 调用 `setAppPipActive` 和 `readClipDataVersion`，即使在 Memo/AI tab 上也会执行，可能触发不必要的重绘。

### 问题5：boot 函数在所有 tab 上执行 refresh
`boot` 函数在键盘加载时调用 `refresh(true, lifecycle)` 获取剪贴板数据，即使当前在 Memo/AI tab 上也会执行，导致不必要的重绘。

### 问题6：回调函数每次渲染都创建新引用
`insertMemoText` 和 `copyMemoText` 作为内联函数每次渲染都创建新引用，导致 MemoPanel 及其子组件不必要地重绘。

### 问题7：boot 函数无条件调用 setLoading(false)
`boot` 函数的 `finally` 块无条件调用 `setLoading(false)`，即使 `loading` 已经是 `false`（`initialLoaded` 为 `true` 时），在 Scripting 框架中可能仍触发重绘。

### 问题8：剪贴板监听器在 Memo tab 上不必要地启动
`scheduleKeyboardMonitor` 在所有 tab 上都会启动，监听器的首次 tick（~350ms）会捕获剪切板并写入 Storage（`bumpClipDataVersion`），可能在框架层面触发重绘。

### 问题9：contextMenu 对象每次渲染都创建新引用
`MemoTile` 组件中 `contextMenu` 使用对象字面量创建，每次渲染都会产生新引用，导致即使组件因其他原因重绘时 SwiftUI 也可能重建 context menu。

## 修复内容

### 1. 添加 activeTabRef
使用 `useRef` 存储最新的 `activeTab` 值，确保 `setInterval` 中能获取到当前标签。

### 2. 优化 setInterval 逻辑
在非剪贴板 tab（Memo、AI）上跳过所有状态更新（`setAppPipActive` 和 `refresh`），但仍更新 `lastSeenClipDataVersion` 以避免切回时触发多余刷新。

### 3. 优化 refresh 函数
在调用 `setItems` 之前，使用 `clipListKey` 比较新旧数据，只有当内容真正变化时才更新状态。

### 4. 优化 boot 函数
- 只在显示剪贴板项目的 tab 上执行 `refresh`
- `finally` 块仅在 `!initialLoaded` 时调用 `setLoading(false)`，避免无意义的重绘
- 只在剪贴板 tab 上调用 `scheduleKeyboardMonitor`

### 5. 稳定回调引用
使用 `useRef` + `useMemo` 创建稳定的 `stableInsertMemoText` 和 `stableCopyMemoText` 回调，避免 MemoPanel 不必要的重绘。

### 6. 优化剪贴板监视器回调
添加了 `shouldRefreshVisibleClipboardItemsRef` 来存储最新的 `shouldRefreshVisibleClipboardItems` 值。

### 7. 切换到剪贴板 tab 时启动监听器
在 `[activeTab]` useEffect 中，切换到剪贴板 tab 时调用 `ensureKeyboardMonitor()` 确保监听器启动。

### 8. MemoTile contextMenu 引用稳定化
- `contextMenu`：使用 `useMemo` 缓存，依赖 `[memo.text, memo.enableSubfields, settings.memoSubfieldSeparators, onInsertText, onCopyText]`
- `onPress`：使用 `useMemo` 缓存回调函数
- `leading`：使用 `useMemo` 缓存 JSX 元素
- 确保 `KeyboardActionTile` 的所有 props 引用稳定，防止 context menu 被重建
